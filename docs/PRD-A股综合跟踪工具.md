## 1. 产品理念(S00–S03 哲学底座)

课程前四篇是「心智模型」而非可计算方法,作为产品的**价值底座**陈述,不功能化(见 §1.2)。

### 1.1 四条核心理念

1. **避险第一,而非盈利**——投资的第一刚需是避险。工具的所有「提醒」本质都是避险动作的工程化。
2. **决策归你自己**——「不要信仰任何人,包括本工具」。工具**诊断**、**提醒**,但**不替你拍板**。全自动决策引擎是 `stock-agent` 的领地,本工具不越界。
3. **市值是虚幻的投影**——交易价格虚构财富总量。所以工具看**估值分位**(PE/PB 在历史中的位置)而非绝对价格;「价格涨了」不等于「便宜了」。
4. **庞氏意识 + 杠杆第一要义**——合规约束下高息必庞氏;非确定性下绝不动杠杆。

### 1.2 避险如何内化进产品

不另起「避险检查器/庞氏雷达」功能模块(那会引入「用户持仓管理」新分支,与诊断工具定位脱节)。避险精神**内化在信号提醒的防守性**里:

- 业绩变脸 → 第一时间预警(成长股风控,S10)
- 跌破 60 日线 → 止盈止损提醒(择时,S13)
- 周期 PB 触历史高位 → 顶部警告(S11)
- 筹码相位转「兑现中段」→ 机构在跑(重远·6 相位)

这些都是**避险动作**,见 §8。

## 2. 产品定位与目标用户

### 2.1 定位:**诊断 + 轻量信号提醒**(只读分析为主)

> 给散户一双**看懂数据的眼睛**。输入标的 → 输出「它现在估值/业绩/择时处于什么历史位置」的诊断,并在关键节点把该看的事推到眼前。**决策你自己做。**

**不做什么**:不出买卖点交易动作、不管你的持仓、不做全自动决策(那是 `stock-agent` 规则引擎的职责)。

### 2.2 目标用户:**自用为主 + 小圈子可分享**

自己深度使用,同时做成可分享的本地看板,方便发给星球同好/朋友。**不做**多用户、账号、云端部署(违反「每个人自己算」的课程精神,且工程量不必要)。

## 3. 三层架构总览

颗粒度由文档方法分布决定(个股 ~70%、指数 ~25%、ETF ~5%),但三层共享同一个心智模型——**价值 / 成长 / 周期 三类分类**。

```
┌─────────────────────────────────────────────┐
│  个股层(主体)        Phase 2               │
│  三类自动判定 · 利润归因 · 避坑四类          │
├─────────────────────────────────────────────┤
│  指数/大盘层(背景)   Phase 1(与ETF并行)  │
│  偏离极值 · 趋势 · 蓝筹vs成长 · 估值开关     │
├─────────────────────────────────────────────┤
│  ETF层(三类分类升级)  Phase 1              │
│  重构 research/ · 通用底盘+差异化主指标      │
└─────────────────────────────────────────────┘
        ↑ 贯穿:统一分类器(分类→选指标)
```

### 3.1 为什么 ETF 先行、个股在后

数据层现状(`fetcher.py`/`store.py` 实测):**个股财务数据几乎为零**,整个财务数据栈要从零建;而 **ETF/指数层数据基本现成或接口已探到**。所以第一阶段吃现成的 ETF + 指数(快赢),个股财务栈作为第二阶段独立排期。框架先抽象对,数据慢慢补。

## 4. 核心心智模型:三类分类

### 4.1 统一分类器

**一处定义「分类 → 选指标」的规则,三层共用**(个股/ETF/指数)。分类器输出 `(主类型, [次类型])`,主类型驱动主指标路径,次类型附加展示。

- **统一的是**:分类框架与「按标签选指标」的逻辑;
- **不统一的是**:判定输入——个股用财务数据**自动算**,ETF/指数用**人工标签 + 行业映射**。

### 4.2 标签:主标签 + 次标签(软分类)

**人工标注为主**(ETF 仅 ~25 只,人工最快最准;数据驱动在 ETF 层判定材料不足),`csrc_industry` 行业映射作**预填建议 + 一致性校验**。

软分类解决「既分红又成长」的混合体(食品饮料/医药)——不再被迫二选一,主类型跑主排名,次类型指标附加展示。呼应 S06「估值/增长/盈利能力要一起看」。

### 4.3 ETF 草案归类(grilling 已认可)

| 类型 | 比较纯的 | 模糊(主+次) |
|---|---|---|
| **价值** | 银行、红利低波 | 家电(主价值/次成长)、食品饮料(主价值/次成长) |
| **成长** | 半导体/AI/软件/机器人/光伏/创新药/创业板/科创50/军工/纳指 | 医药(主成长/次价值) |
| **周期** | 煤炭/有色/化工/证券/养殖/电力/地产/汽车/黄金 | — |

## 5. ETF 层设计(Phase 1)

### 5.1 架构:通用底盘 + 差异化主指标

> grilling 关键修正:**不是**「三套完全独立指标」,**也不是**「全部塞通用底盘 + 差异化解读」。而是**融合**:通用因子三类共享,主指标按类型独立(周期用 PB 不搞 PE 反转,清晰优先)。

### 5.2 三类指标规格

| | **通用底盘**(所有 ETF 共有) | **差异化主指标**(按类型) |
|---|---|---|
| **价值** | 筹码相位 + 趋势 | 股息率(Phase1 稀疏,无则标「数据不足」)+ PE 分位 + 分红稳定 |
| **成长** | 筹码相位 + 趋势 | 业绩预告聚合(yoy/预增预减)+ PE 分位 |
| **周期** | 筹码相位 + 趋势 | **PB 历史分位**(替 PE);Phase1 板块 PB 不可得,**临时降级行业 PE 反向**(标注降级),Phase2 成分股加权补真 PB |

- **筹码相位**(重远·6 相位非单调:兑现中段最空、深回撤+卖盘枯竭=见底最看多):三类通用,**必保留**。回答「机构现在在干什么」。
- **趋势**(close vs MA,60 日):三类通用。
- **PE 分位**:价值/成长用;**周期换成 PB**(S11:周期看 PB 不看 PE,PE 在周期顶会失真)。
- **成长不凑 PEG**:ETF 无「未来预期 G」,用业绩三件套(预告聚合 + PE 分位 + 趋势),PEG 留给有完整财务数据的个股层。

### 5.3 原地重构 `research/`

`scoring.py` 从「一刀切三因子(估值/筹码/趋势)」改写为「按类型走差异化主指标 + 通用底盘」。`earnings.py`(业绩预告聚合)归入成长型因子(现成)。纯函数 `valuation_score`/`trend_score`/`chip_score` 尽量复用。

### 5.4 ETF 层数据缺口(接口已探到)

| 缺口 | akshare 接口 | 用途 |
|---|---|---|
| 周期型 PB | `stock_index_pb_lg` | ⚠️ **只支持宽基,不支持中证煤炭等行业指数** → Phase1 周期型**临时降级行业 PE 反向**(cninfo),Phase2 用成分股 `stock_zh_valuation_baidu` 加权补真 PB |
| 价值型股息率 | `fund_etf_dividend_sina` | ⚠️ **覆盖差**(3 个 ETF 仅 1 个返回数据)→ 容忍稀疏,无则标「数据不足」;Phase2 用成分股 `stock_history_dividend` 加权补全 |

## 6. 指数择时层设计(Phase 1,与 ETF 并行)

基于 S12–13。**数据极轻**(5 个宽基日线),是所有诊断的背景,提前到第一阶段。

### 6.1 五件套(全要)

| # | 功能 | 文档 | 说明 |
|---|---|---|---|
| **① 偏离极值曲线**(核心可视化) | S13 | 价格+60日线主图 / 偏离度副图(价格/均线−1)+ 历史极值水平线(±X%),当前偏离高亮。**先看见极值在哪,才能判断是否接近、该不该套利** |
| ② 60日线趋势状态 | S12-13 | 每个宽基在上/下、均线趋势上/下 → 左/右侧、趋势 vs 震荡(震荡市关闭趋势信号) |
| ③ 蓝筹 vs 成长对比 | S13 | 上证50(蓝筹) vs 创业板/中证500/科创50(成长) → **仓位倾向**(都上→偏成长,都下→偏蓝筹,相反→偏向上的) |
| ④ 大盘估值开关 | S13 | 沪深300 估值位置(PE 分位/3000 点)→ 决定趋势信号**敏感度**(低位更激进、高位更保守) |
| ⑤ 有效突破/跌破 | S13 | 指数级 E1/E2/F1,5–6 档确定性梯度 |

### 6.2 数据

- 5 个宽基(上证50/沪深300/中证500/创业板/科创50)日线:`index_zh_a_hist`,写入 `daily_prices`(TEXT symbol 已支持)或新表 `index_daily`。
- ④ 估值开关数据(技术尽调已确认):**沪深300 PE** via `stock_index_pe_lg`(5169 行,2005 起,`滚动市盈率`)+ **全市场 PB** via `stock_a_all_pb`(2005 起,作大盘估值背景)。创业板指/科创50 不在支持集,不强求。

## 7. 个股层设计(Phase 2 预览)

Phase 2 触发条件:Phase 1 跑通 + 个股财务数据栈就位。

### 7.1 能力清单

- **三类自动判定**:用财务数据算(增速高→成长、股息率高 PE 低→价值、利润历史波动大→周期)。
- **利润来源归因**(S07 茅台拆解):给定持仓期,拆解**业绩/分红/估值**各自贡献——课程指定的「真正获得投资体感」的功能。
- **避坑四类**(S08):公告时间差校正、两年复合增速、异常高增速(最小值分母)、业绩预告/快报/正式报/公司自带下季预测跟踪。
- **戴维斯双击/双杀**(S10):个股级业绩变脸预警 + 抄底估值测算。
- **极值偏离套利**(S13):个股股价/均线偏离。

### 7.2 个股财务数据栈(从零建)

数据层现状:营收/归母净利/扣非/ROE/ROA/毛利率/分红/EPS/PE/PB **全无**;个股日线因 `_szsh_prefix` bug 实际拿不到(SSE 主板股全坏)。

**金矿已验证**:`stock_zh_valuation_baidu` 确认可拿 PE(TTM)/PE(静)/PB/市现率/总市值,**全市场、IPO 起**——个股 PE/PB **免从财报硬算**。**但无股息率**(需从 `stock_history_dividend` derive)。个股日线修 `_szsh_prefix`(`6xx/68x/9xx→sh`)。

## 10. 技术架构

```
stockagent/

├── engine/
│   ├── indicators.py     ← 复用 (percentile_rank / sma → 估值分位/趋势)
│   └── signals/share_flow← 复用 (_classify_state → 筹码6相位)
├── research/          ← 重构:ETF层三类分类
│   ├── scoring.py        (一刀切三因子 → 按类型走差异化主指标 + 通用底盘)
│   └── earnings.py       (归入成长型因子,现成)
├── tracker/           ← 🆕 新工具核心
│   ├── classifier.py     (三类标签 主+次; 统一分类器入口)
│   ├── diagnose.py       (个股/ETF/指数 三层诊断)
│   ├── alerts.py         (信号提醒规则库 —— §8 心脏,双通道同源)
│   └── dashboard.py      (生成交互式 HTML 看板)
├── notify/            ← 复用 (微信推送信号提醒)
└── report/llm_writer  ← 复用 (提醒文案,延续晨报风格)
```
