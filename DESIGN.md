# stock-agent · 产品设计文档

> 一句话：一个**个人用**的 A股 板块轮动 ETF 决策助手——用规则引擎替你执行"满仓最强 3 个板块 + 大盘不好就避险 + 单仓止损"的纪律，用大模型把信号翻译成每天早晨微信/飞书上的说人话报告，并追踪你的执行自律度。
> 状态：需求/设计 grilling 已完成（16 支全走通），可作为 V1 实施依据。

---

## 一、产品定位

- **用户**：仅作者本人（V1 不考虑多用户）。
- **胜利条件**：让作者更**自律**地执行板块轮动——在正确的时间处于正确的 A股 ETF 中，**严格控制回撤**，并**清楚地解释原因**；每个交易日早晨在**微信/飞书**收到一份报告。
- **产品哲学**：大部分早晨的报告是"**维持持有，无需操作**"——挡住交易欲，这本身就是核心价值。

---

## 二、系统架构

```
                 ┌─────────────────────────────────────────────┐
   AkShare/Baostock│ ① 数据层（15:30 收盘后批量拉取）              │
                 │   前复权日线 OHLCV + 交易日历 → SQLite         │
                 └──────────────────────┬──────────────────────┘
                                        ▼
   ┌────────────────────────────────────────────────────────┐
   │ ② 规则引擎（纯函数，有单测；决策归此层）                  │
   │                                                        │
   │   保护层（每日收盘跑）            轮动层（每周五收盘跑）    │
   │   ├─ 大盘择时：沪深300 跌破       ├─ 趋势门槛：价>60日线    │
   │   │  120日线 → 100%转货币ETF       ├─ 动量分：0.2×20日      │
   │   ├─ 单仓位 -8% 移动止损          │  +0.3×60日+0.5×120日    │
   │   └─ 优先级：止损>轮动>择时       └─ 取前 K=3，等权          │
   │                                                        │
   │   → 输出结构化信号 JSON（动作/持仓/盯防/状态）            │
   └──────────────────────┬─────────────────────────────────┘
                          ▼
   ┌────────────────────────────────────────────────────────┐
   │ ③ 大模型层（解释归此层，不发明数字、零预测）              │
   │   JSON 信号 → 三段式中文早晨报告                          │
   └──────────────────────┬─────────────────────────────────┘
                          ▼
   ┌────────────────────────────────────────────────────────┐
   │ ④ 推送层（Notifier 接口，通道无关）                       │
   │   企业微信群机器人(主) / 飞书(格式最强) / PushPlus(备)     │
   │   早 8:30 推送；失败必告警；幂等自愈补跑                   │
   └──────────────────────┬─────────────────────────────────┘
                          ▼
                 你（人工执行）+ 自律度对账
```

**数据流时序（每个交易日）：**
1. **15:30 后**：数据层拉当日收盘日线入库（非交易日跳过）。
2. **引擎**：保护层每日跑（择时/止损）；轮动层仅周五跑。
3. **次日 8:30**：基于前日收盘信号，大模型写报告 → 推送。
4. **9:30 开盘**：你按报告"今日动作"在券商App下单；点"已执行"反馈。

## 三、策略规格（引擎规则）

### 大盘择时层（每日，最高优先级之一）
- 基准：**沪深300（510300）**。
- 风险关：日收盘 **< 120日半年线** → 3 个板块 ETF **全部清仓**，100% 转 **货币ETF（华宝添益 511990）**。
- 风险开：日收盘 **> 120日线** → 恢复 K=3 轮动持仓。
- 对称单线触发，无二次确认（回测显示太抖再加补丁）。

### 轮动层（每周五收盘）
- **趋势门槛**：板块ETF 收盘价 **> 60日均线** 才进候选池。
- **动量分**：`0.2×(20日涨幅) + 0.3×(60日涨幅) + 0.5×(120日涨幅)`。
- **选股**：候选池按动量分降序，取**前 K=3**，**等权**（各 1/3）。
- 排名滑出前 3 → 下周一开盘轮动换掉。

### 单仓位止损层（每日）
- **-8% 移动止损**：自持仓期间**最高价**回撤超 -8% 触发。
- 当日收盘判定 → **次日开盘执行**；资金暂存货币ETF，等周五轮动补仓。

### 引擎优先级硬规则
**止损 > 轮动 > 择时**（同日冲突时按此顺序）。

### 可调参数（一律回测扫描，不拍死）
K（2/3/5）、动量窗口与权重、门槛均线（60/120日）、止损%（5/8/10/12）、择时均线（60/120/200日）。

## 四、数据规格

- **源**：AkShare（主）+ Baostock（兜底）；带重试 + 本地库降级。
- **复权**：**前复权**（实盘/引擎计算）；回测用**后复权**（避前视）。
- **存储**：本地 **SQLite**（date+symbol 唯一键）。
- **范围**：~25 标的（~20 板块ETF + 沪深300 + 货币ETF），历史 ≥5 年。
- **更新**：每交易日 15:30 后批量；**V1 无需盘中数据**。
- **校验**：完整性检查（无缺口/空值、复权价合理、量>0），脏数据拒入。
- **交易日历**：AkShare，节假日不跑。

---

## 五、回测规格（自研 pandas 向量化）

**七宗罪治法：**
| 陷阱 | 治法 |
|---|---|
| 前视 | T日收盘信号、T+1开盘成交；后复权 |
| 幸存者偏差 | V1 用当前池（偏差可控），point-in-time 留 V2 |
| T+1 | 强制：当日买不可卖 |
| 成本/滑点 | 单边 ~0.1%（ETF无印花税） |
| 成交价 | T+1 开盘 ± 滑点 |
| 过拟合 | 少参数 + walk-forward + 固定周五（不优化周几） |
| 无基准 | 对比沪深300 buy&hold + 60/40 |

**硬门槛（决策门）**：在**最大回撤显著低于**沪深300 buy&hold 前提下，年化**不输**沪深300。**达不到 → 策略无价值，停下调参或承认该 buy&hold。**
**指标**：年化、最大回撤、Calmar、Sharpe、换手率、胜率。

## 六、报告与推送规格

- **报告结构**：三段式（🟢今日动作+理由 / 📊组合状态·大盘·盯防 / 🔍简评）。
- **原则**：动作优先 · 长短自适应（无操作则极简）· **零预测**（绝不写"预计上涨"）· 不堆全排名 · 标数据日期。
- **推送**：Notifier 接口；企业微信(主)/飞书(格式最强)/PushPlus(备)；早 **8:30**；V1 纯文本（企业微信不支持表格，飞书可用卡片）。
- **禁用**：itchat 类个人微信外挂（封号风险）。

---

## 七、执行与自律度规格

- **V1 执行**：纯建议 + **人工执行**（报告=指令单，券商App下单）。
- **自律度对账（核心闭环）**：系统维护"目标持仓"；你 tap"已执行"或每周手输实际持仓；系统算**偏离度**并在报告展示（"本周自律度 X%"）。不对账=放弃度量自律。
- **自动交易**：推迟 V2+，前置四条件（回测+实盘验证、QMT资格、kill-switch、心理接受）。
- **shadow 纸盘期**：实盘前只读跟踪 **4–8 周**再投真钱。

## 八、技术与运行规格

- **语言**：Python；**最小依赖**（pandas/numpy/akshare/baostock/requests/LLM SDK/pytest）；不套重框架。
- **结构**：`config / data / engine / backtest / report / notify / scheduler / tests`。
- **配置/密钥**：参数+ETF池 YAML（入git）；key/webhook .env（不入git）。
- **测试**：引擎规则（动量/择时/止损）必须有单测。
- **LLM**：便宜国产模型（GLM-Flash / DeepSeek），可热插拔。
- **运行**：V1 **本地 Windows 任务计划程序**；购机后迁云（SQLite 可分钟级迁移）。

---

## 九、调度与可靠性规格

- **交易日历门禁**：非交易日不跑不推。
- **幂等自愈补跑**：任务"今天做了吗？没做就做"；数据缺自动回填；报告漏发唤醒补发（标"迟发"，过开盘标"指令时效已过"）；幂等不重复推/不写脏。
- **失败必告警**：数据连续失败/引擎异常/LLM失败 → 推"⚠️系统异常"，**绝不静默**（静默=漏信号=亏钱）。
- **心跳死亡开关**：迁云后补（9:00无报告即告警）；本地版接受"关机则静默"盲区，用开机自愈兜底。
- **可观测性**：结构化日志 + 报告含数据健康标记。

## 十三、A股硬约束（设计时已遵守）
- **T+1**：当日买次日才能卖 → 无日内轮动；止损次日开盘执行。
- **涨跌停**：个股±10%、双创±20%；ETF一般不受限但看流动性。
- **交易时段**：9:30–11:30 / 13:00–15:00；报告须 9:30 前基于前日收盘生成。
- **牛短熊长**：必须有大盘择时，否则熊市必腰斩。

## 十四、M2 回测实证结论（2026-07 实跑，真实数据 sina_raw）

在 2021-01 至 2026-07（A股典型"牛短熊长+风格剧震"样本）上：
- **基准沪深300 buy&hold**：年化 -1.8%、最大回撤 -45%。
- **V1 动量轮动策略（24 组参数扫描）**：**0/24 通过决策门**。最佳组合（K=3 / 大盘120日线 / 止损12% / 趋势门槛60日线）年化 -3.7%、回撤 -42%——**回撤略低于基准但收益仍跑输**，未达"更低回撤下不输基准"的硬门槛。
- **诊断**：① 大盘择时层确实生效（54% 交易日 risk_off，2022 大熊市 -5.6% 远好于基准）② 失血主因是 risk_on 时段轮动进随后反转的动量板块 + 止损反复被扫 + 12× 年换手成本。**与"A股动量弱、均值回归强"的已知特性一致。**

**结论**：V1 基线策略**如实未通过决策门** → 按 DESIGN 规则**不应据此上实盘**。系统/回测器/决策门本身正确履职（证伪有效）。策略需迭代（见 V2 待办：相对强度、资金流确认、**加入均值回归/反转成分**、降换手），用户应在策略通过决策门后再启动 shadow/实盘。M3–M5 的报告/推送/调度/对账/shadow 基础设施先行建好，策略一达标即可接入。

**数据说明**：当前为 sina_raw（不复权）；eastmoney 解封后重跑 `update_data.py` 升级 hfq，绝对收益会略升（分红回补），但相对结论不变（策略与基准同源）。

## 十五、V2.1：可插拔信号 + 均值回归因子（2026-07）

**做了什么**：把"排序信号"抽成可插拔接口（`engine/signals/`，注册表 `get_signal`，配置 `rotation.signal.name` 切换），新增**均值回归信号**（买长期上升中的超跌回调：`close>MA(long_ma)` 且 `RSI<oversold`，打分 `oversold−RSI`），并把信号作为维度接入 sweep 与 walk-forward。新增 RSI 指标。momentum 信号与全部 V1 测试保留为零回归基线。

**样本外判决（walk-forward，训练 2021-23 / 测试 2024-26）**：

| 信号 | 样本外年化 | 最大回撤 | Sharpe | 决策门 |
|---|---|---|---|---|
| momentum（训练选参） | -15.4% | -53% | -0.12 | FAIL（过拟合崩盘） |
| **reversion（训练选参）** | **+5.5%** | **-9.4%** | **0.58** | FAIL（收益未超基准 +15.3%） |
| 沪深300 buy&hold | +15.3% | -16.3% | — | — |

**结论**：
- reversion 是**第一个样本外站得住的信号**——不过拟合、正收益、回撤仅基准一半。它**样本外碾压 momentum**（+5.5%/-9.4% vs -15.4%/-53%）。
- 但仍 FAIL 严格决策门：2024–26 沪深300 是 +15% 强反弹，reversion 用"低收益换低风险"，回报追不上强牛市。
- **定性**：reversion 是真正的**防御型信号**（熊市少亏/牛市小赚/回撤极低），完美契合 DESIGN 路线 A（纪律与资本保全工具），但**不是 alpha 引擎**。
- 严格决策门"收益≥基准"对防御型策略可能过苛；对 reversion 更公平的标尺是 Sharpe/Calmar + 多窗口 walk-forward。

**下一步选项**：① 接受路线 A，把 reversion 设为默认信号、按"低回撤纪律工具"定位进 shadow；② V2.2 加资金流/相对强度追 alpha；③ 对 reversion 做多窗口 walk-forward 验证稳健性。

### V2.1-step-B：多窗口 walk-forward 稳健性验证（2026-07）

对 reversion 做**4 个年度 expanding 窗口**样本外测试（训练锚定 2021-01-01，逐年测试 2023/2024/2025/2026）：

| 测试年 | 策略年化 | 策略回撤 | 基准年化 | 基准回撤 | 门 | 回撤<基准 |
|---|---|---|---|---|---|---|
| 2023 | +0.01% | -0.08% | -11.9% | -19.9% | PASS | ✓ |
| 2024 | +0.01% | -0.15% | +17.3% | -12.4% | fail | ✓ |
| 2025 | +18.2% | -9.4% | +22.6% | -10.1% | fail | ✓ |
| 2026 | +0.03% | -0.05% | +0.27% | -9.9% | fail | ✓ |

汇总：gate PASS **1/4**；平均年化 **+4.57%** / 平均回撤 **-2.41%**；**回撤<基准 4/4**；收益≥基准 1/4。

**判读**：
- **稳健（非过拟合）**：4 个窗口训练选出的最优参数**完全相同**（K=3/regime120/rsi14/oversold30/long250），参数不随训练窗口漂移。

## 十九、V3.0：value_flow 信号（买低位 + 聪明钱 + 自适应追踪止损）— 2026-07

**动机**：承接十八的"重构 share_flow 为 filter"方向——把**聪明钱（份额趋势）当避险/确认过滤器**，叠加**历史低位（价值）入场**与**双层止损**，做第 5 个可插拔信号。区别于前 4 个的价量/份额 ranking，value_flow 是"**低位+机构买入+耐心持有**"的 buy-and-hold，非周度轮动。

**入场**：`ACCUMULATING`（机构净买入）**且** 价格分位 `< entry_percentile`（历史低位）**且** `price_stabilized`（近期未创新低=止跌企稳）。
**持有/退出**（双层）：
- Layer 1（始终生效，价格即真理）：相对**入场价**跌幅 `> entry_stop` → 出。"我以为是低位，再跌 15% 就认错"。不依赖份额状态，亏损不死扛。
- Layer 2（仅保护利润）：相对**峰值**的 adaptive trailing，距离按份额状态：`ACCUMULATING → 3×ATR`（让利润奔跑）、`STABLE → 2×ATR`、`DISTRIBUTING → 1×ATR`（机构走=收紧）。仅当峰值高于入场价（曾盈利）才激活。
- `min_hold_days`（60 天）防过早止损；`SUPER_STICKY=True`：持仓免疫周度轮动，**只有 check_exits 能踢出**。

**单窗口样本外**（walk-forward，训练 2021-23 / 测试 2024-26）：

| 信号 | 样本外年化 | 回撤 | Sharpe | 门 |
|---|---|---|---|---|
| reversion（旧最优） | +5.5% | -9.4% | 0.58 | FAIL |
| **value_flow** | **+5.86%** | **-7.41%** | **0.79** | FAIL（收益<基准） |

value_flow 在**风险调整**上全面超越 reversion（更高收益、更低回撤、更高 Sharpe），但仍 FAIL 严格决策门（收益 < 沪深300 +15.51% 强牛市）——**定性仍是防御/保本型**，回撤条件已满足（-7.41% < -16.25%）。

### V3.0-step-B：多窗口 walk-forward 稳健性验证（2026-07）

对 value_flow 做**4 个年度 expanding 窗口**样本外测试（训练锚定 2021-01-01，逐年测试 2023/2024/2025/2026）——给其与 reversion（十五 step-B）同等的稳健性检验：

|---|---|---|---|---|---|---|

汇总：gate PASS **2/4**；平均年化 **+2.33%** / 平均回撤 **-5.38%**；**回撤<基准 4/4**；收益≥基准 2/4。

**判读**：
- **稳健（非过拟合）**：4 窗口训练选出的参数**几乎完全相同**（K=3 / regime=200 / pct=0.20 / trail_d=1.0 全 4 窗；trail_a=2.0 出现 3 次、3.0 出现 1 次）。参数不随训练窗口漂移 → 信号非样本内巧合。

- **回撤控制 4/4 全胜**：每个窗口策略回撤都低于基准（最低 -1.32%，最高 -9.20%），完美契合路线 A"资本保全/纪律工具"定位。

**结论**：value_flow 是**目前最稳健的信号**——多窗口参数稳定、回撤全面碾压基准、gate PASS 数翻倍 reversion。**推荐设为默认信号进 shadow 观察**（已切换 `rotation.signal.name: value_flow`）。仍非 alpha 引擎（牛市跑输基准），严格决策门未过；但其风险调整表现与多窗口稳健性已达到"可小额实盘验证"的门槛。

**元规律（第五次确认）**：A股 稳定 alpha 极难，但"**低位+聪明钱确认+严格止损**"的组合在样本外展现出最一致的**防御+择时**价值——印证十八的判断：份额/聪明钱维度更适合做**确认/过滤**而非追买 ranking。继续追 alpha 的方向仍是接入**北向/持仓明细**等更多聪明钱数据，或**多信号组合**（ensemble 防御型 + 进攻型）。
